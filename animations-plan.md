# Animations Plan

## Контекст

- Базовая стилевая интеграция уже собрана в ветке `codex/main-integration`.
- Новая анимационная ветка должна идти поверх этой базы.
- Текущий стек уже подходит для следующего этапа: `React + Framer Motion + CSS variables`.
- Простые появления и hover уже частично есть, но пока это не система, а набор разрозненных эффектов.

## Главная цель

Сделать анимации частью всего сайта, а не только карточек проектов:

- усилить первое впечатление
- добавить ощущение премиальности
- улучшить storytelling проектов
- не сломать производительность и мобильную адаптацию
- не уйти в кринж, перегруз и визуальный шум

## Что можно реализовать уже сейчас

Это можно делать без переделки админки и без расширения модели данных.

### P0. Общая система анимаций для всего сайта

Нужно ввести единый набор motion-паттернов:

- page enter / page exit
- reveal on scroll
- stagger для списков и карточек
- hover для карточек, изображений и CTA
- микроанимации кнопок
- поддержка `prefers-reduced-motion`

Зона применения:

- header
- mobile menu
- footer
- homepage
- category pages
- project pages
- services
- about
- blog
- contact

Почему это приоритет №1:

- это даёт целостность всему сайту
- это можно внедрять быстро и безопасно
- это создаёт основу для всех остальных анимаций

### P0. Hero “Погружение в интерьер”

Реализовать в первую очередь:

- fullscreen image hero
- мягкий zoom-in фона `scale(1 -> 1.05)`
- staged reveal текста
- лёгкий fade/translate для CTA
- более кинематографичную overlay-подложку

Применение:

- homepage hero
- hero в project page
- при необходимости hero отдельных category pages

Что брать сейчас:

- крупный заголовок 2–3 строки
- минимальный текст
- CTA
- фото вместо 3D

Что пока не брать:

- тяжёлый 3D background
- сложный WebGL
- полноэкранные видео на всех страницах подряд

### P0. Before / After

Это самый важный продающий блок. Его нужно переделать полноценно.

Что должно быть:

- плавное сравнение без дёрганий
- нормальная работа мышью и свайпом
- подписи `Before / After`
- аккуратная линия разделения
- мягкая анимация ручки
- лёгкий highlight при hover / touch
- стабильная работа на mobile

Что важно исправить в текущей реализации:

- логика пересчёта положения сейчас хрупкая
- поведение привязано к raw DOM query внутри события
- визуально блок выглядит техническим, а не премиальным
- сейчас нет ощущения “перетекания”

Итоговый уровень:

- премиальный interactive compare block
- reusable для project pages и отдельной before/after page

### P0. Карточки вместо сухого текста

Не только product card. Нужно системно перейти к card-like presentation там, где это уместно.

Сразу можно усилить:

- project cards
- category cards
- blog cards
- service previews
- featured blocks на homepage
- press/testimonial preview blocks

Что добавить:

- hover elevation
- image zoom
- overlay fade
- иконка / стрелка / preview cue
- stagger reveal при входе секции

### P0. Микроанимации интерфейса

Внедрить по всему сайту:

- кнопки
- ссылки в меню
- mobile menu
- dots / controls
- lightbox
- CTA blocks

Типы:

- opacity
- y-shift
- scale
- blur reveal очень умеренно

## Что лучше делать следующим этапом

Это уже следующий приоритет после базовой системы.

### P1. Scroll storytelling для project pages

Формат кейса как истории:

1. проблема
2. идея
3. процесс
4. результат

Что можно сделать на текущем фронте:

- sticky media + смена текста при скролле
- zoom / parallax у изображения
- активные шаги истории

Что ограничивает:

- для хорошего результата нужны более структурированные блоки проекта
- часть кейсов потребует новой схемы контента в админке

Вывод:

- можно сделать фронтовой прототип уже сейчас
- полноценная система для всех проектов лучше во второй фазе

### P1. Светлая тема с переключателем

Это нужно, но не как “быстрый эффект”.

Что потребуется:

- ревизия всех CSS variables
- нормальная тема для текста, фонов, границ, overlay
- хранение выбора пользователя
- адаптация карточек, hero, footer, mobile menu, lightbox

Вывод:

- делать после унификации motion-system
- это отдельная подзадача уровня whole-site theming

### P1. Более живые медиа-галереи

Можно добавить:

- stagger появления изображений
- красивый open transition в lightbox
- смену активного изображения без тяжёлых контролов
- мини-превью сбоку или снизу

Это особенно полезно для project page.

## Что лучше вынести на потом

Это уже не “следующий спринт”, а отдельная future-phase.

### P2. Видео-фоны

Имеет смысл только точечно:

- homepage hero
- selected premium projects

Почему не сразу:

- вес
- autoplay/mobile ограничения
- отдельная подготовка видео
- нагрузка на performance

### P2. Настоящий parallax / pseudo-3D / layered depth

Можно сделать красиво, но осторожно.

Риски:

- лаги на слабых устройствах
- неочевидная польза по сравнению с хорошим motion design
- сложнее поддерживать

Рекомендация:

- сначала построить сильную 2D motion-систему
- потом точечно добавлять глубину

### P2. Lighting mode: день / вечер / тёплый / холодный

Идея сильная, но это уже semi-editorial experience.

Для хорошей реализации нужны:

- разные подготовленные изображения или цветовые версии
- отдельная логика переключения
- решение, кто этим управляет: код или админка

Вывод:

- как спец-фича для отдельных top projects
- не как первая общесайтовая задача

### P2. Glassmorphism / blur backgrounds

Можно использовать, но только акцентно:

- floating cards
- CTA overlays
- menu layers

Не стоит превращать в основной стиль сайта.

## Приоритеты по запуску

### Этап 1. Делать сразу

1. Общая motion-system для всего сайта
2. Premium hero с zoom-in и staged text
3. Полный рефактор `Before / After`
4. Hover и reveal для всех карточек и списков
5. Микроанимации header / menu / buttons / CTA

### Этап 2. Делать после этого

1. Scroll storytelling для отдельных кейсов
2. Улучшенные проектные галереи
3. Светлая тема с переключателем

### Этап 3. Делать позже

1. Видео-фоны
2. Pseudo-3D / advanced parallax
3. Lighting mode
4. Glassmorphism как доп-слой, не база

## Техническая стратегия

### Сейчас

- использовать уже подключённый `Framer Motion`
- собрать motion presets в одном месте
- минимизировать inline animation logic
- вынести reusable patterns в shared helpers/components

### Потом

- добавить theme switch architecture
- при необходимости расширить project blocks
- отдельно продумать admin-controlled animation variants
